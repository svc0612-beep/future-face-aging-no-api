최종 모델 조합

현재 기준 최종 방향: 나이 예측은 ConvNeXt-Tiny를 낮은 학습률로 추가 튜닝한 fine-tuned 모델, 성별 분류는 MobileNetV3-Small, 미래 얼굴 시뮬레이션은 SAM을 사용합니다. 단, 미래 얼굴 이미지는 실제 미래 예측이 아니라 AI 기반 재미용 시뮬레이션으로 안내해야 합니다.

데이터/전처리 용어

학습 데이터 준비와 모델 입력을 만들 때 나온 용어입니다.

UTKFace
유티케이페이스
데이터셋

파일명에 나이, 성별, 인종 정보가 들어 있는 공개 얼굴 데이터셋입니다.

  • 프로젝트에서 나이 예측 학습 데이터로 사용
  • 전체 24,106장 중 정상 파싱 24,102장
  • 인종/조명/품질 편차가 있어 실제 웹캠 입력과 차이가 날 수 있음
AI Hub 안면 인식 에이징 데이터
에이아이 허브
데이터셋한국인 중심

한국인 얼굴 기반의 에이징 이미지 데이터입니다.

  • 프로젝트에서 가장 중요한 한국인 얼굴 데이터
  • 최종 매핑 45,200개 성공
  • 이미지와 JSON 라벨 매핑 과정이 필요했음
Metadata
메타데이터
CSV

이미지 경로, 나이, 성별, 데이터셋 출처 같은 학습 정보를 표로 정리한 파일입니다.

  • `combined_metadata.csv`: AIHub + UTKFace 통합
  • `final_metadata.csv`: 학습 split 생성 후 기준 파일
  • 모델 학습은 이 CSV를 기준으로 이미지와 라벨을 읽음
Train / Validation / Test Split
트레인 / 밸리데이션 / 테스트
데이터 분리

데이터를 학습용, 검증용, 최종 평가용으로 나누는 작업입니다.

  • Train: 모델이 직접 학습하는 데이터
  • Validation: 학습 중 best 모델 선택에 사용
  • Test: 최종 성능 평가에 사용
Group Leakage
그룹 리키지
주의

같은 사람이 Train과 Test에 동시에 들어가 모델 성능이 실제보다 좋아 보이는 문제입니다.

  • AI Hub는 파일명 첫 토큰을 group_id로 사용
  • 여러 split에 섞인 group 수 0으로 확인
  • 신뢰 가능한 평가를 위해 꼭 막아야 하는 문제
Dataset / DataLoader
데이터셋 / 데이터로더
PyTorch

이미지와 라벨을 읽어 모델 학습용 batch로 만들어주는 PyTorch 구성요소입니다.

  • 프로젝트 테스트 batch shape: [16, 3, 224, 224]
  • 이미지를 Tensor로 바꾸고 정규화해 모델에 입력
Face Crop / Alignment
페이스 크롭 / 얼라인먼트
정확도 개선

사진에서 얼굴 영역만 자르고 눈, 코, 입 위치를 기준으로 얼굴을 정렬하는 전처리입니다.

  • 나이 예측 오차를 줄이는 핵심 전처리
  • SAM 미래 얼굴 생성에도 매우 중요
  • 얼굴이 작거나 기울어져 있으면 결과가 흔들림
dlib Landmark Predictor
디립 랜드마크 프리딕터
얼굴 정렬

눈, 코, 입, 턱선 등 얼굴의 68개 기준점을 찾는 도구입니다.

  • SAM 입력 사진 정렬에 사용
  • `shape_predictor_68_face_landmarks.dat` 가중치 필요

학습 용어

Epoch
에포크
학습 반복

전체 학습 데이터를 모델이 한 번 모두 본 단위입니다.

  • Epoch가 늘면 모델이 데이터를 더 많이 학습함
  • 너무 많으면 과적합이 생길 수 있음
Batch Size
배치 사이즈
학습 설정

한 번에 GPU로 넣는 이미지 개수입니다.

  • Batch size가 크면 VRAM을 많이 사용
  • 현재 MX450 2GB 환경에서는 작은 batch가 필요
Learning Rate
러닝 레이트 / 학습률
학습 설정

모델이 한 번 업데이트될 때 얼마나 크게 움직일지 정하는 값입니다.

  • 너무 크면 학습이 흔들림
  • 너무 작으면 학습이 느림
Early Stopping / Patience
얼리 스탑핑 / 페이션스
과적합 방지

검증 성능이 더 좋아지지 않으면 학습을 일찍 멈추는 방식입니다.

  • Patience 2는 2번 연속 개선이 없으면 중단
  • 시간과 GPU 자원을 아끼는 데 중요
Pretrained / Fine-tuning
프리트레인드 / 파인튜닝
전이학습

이미 대규모 데이터로 학습된 모델을 가져와 우리 데이터에 맞게 다시 학습하는 방식입니다.

  • 처음부터 학습하는 것보다 빠르고 안정적
  • 나이 예측 모델은 사전학습 가중치 사용 시 성능이 크게 좋아짐
Low Learning Rate Fine-tuning
로우 러닝 레이트 파인튜닝
재학습 결과

이미 좋은 성능을 낸 ConvNeXt-Tiny 모델을 크게 흔들지 않도록 낮은 학습률로 조금 더 다듬은 추가 학습입니다.

  • 이번 추가 튜닝 학습률: 1e-5
  • 첫 번째 추가 epoch에서 Validation MAE 5.6993세 기록
  • 두 번째 추가 epoch는 5.7009세로 더 좋아지지 않아 첫 번째 best 모델 유지
  • Test MAE는 5.7122세에서 5.4106세로 개선
Checkpoint / Best / Last
체크포인트 / 베스트 / 라스트
모델 파일

학습된 모델 가중치를 저장한 파일입니다.

  • Best: 검증 성능이 가장 좋았던 모델
  • Last: 마지막 epoch까지 학습한 모델
  • 앱에서는 일반적으로 Best 모델 사용
CUDA / GPU / VRAM
쿠다 / 지피유 / 브이램
하드웨어

NVIDIA GPU를 이용해 딥러닝 학습과 추론을 빠르게 수행하는 환경입니다.

  • 현재 GPU: NVIDIA GeForce MX450
  • VRAM: 2GB라 생성 모델 실행에 제약이 큼
  • SAM은 실행되지만 느리고 OOM 위험이 있음
AMP
에이엠피
성능 최적화

Automatic Mixed Precision의 약자로, 일부 연산을 낮은 정밀도로 처리해 속도와 메모리를 절약하는 기법입니다.

Pipeline
파이프라인
작업 흐름

순서대로 자동 실행되는 작업 묶음입니다.

  • 이번 프로젝트에서는 기존 모델 평가 → 추가 학습 → 다시 평가 → 그래프 생성 → 용어/보고서 정리를 순서대로 실행하는 흐름을 뜻했습니다.
  • 사람이 매번 명령을 하나씩 치지 않아도 정해진 순서대로 다음 작업이 이어집니다.
  • 단, 사용자가 직접 진행 상황을 볼 수 있도록 별도 진행률 페이지를 두는 것이 중요합니다.
Training / Tuning / Fine-tuning
학습 / 튜닝 / 파인튜닝
핵심 구분

모델을 만들거나 개선할 때 자주 헷갈리는 세 가지 용어입니다.

  • 학습: 모델을 처음부터 또는 크게 훈련시키는 것입니다.
  • 튜닝: 이미 학습된 모델을 목적에 맞게 조금 더 조정하는 것입니다.
  • 파인튜닝: 튜닝 중에서도 기존 가중치를 가져와서 새 데이터나 새 목표에 맞게 이어 학습하는 방식입니다.
  • 이번 나이 예측 개선은 ConvNeXt-Tiny 기존 best 가중치를 가져와 낮은 학습률로 이어 학습한 파인튜닝입니다.

평가 지표 용어

Loss
로스
학습 지표

모델이 학습 중 얼마나 틀렸는지를 수학적으로 계산한 값입니다.

  • 낮을수록 학습 목표 기준으로 좋음
  • 작업마다 의미가 달라 서로 다른 모델 종류끼리 직접 비교하면 안 됨
  • Train Loss는 낮은데 Val Loss가 오르면 과적합 가능성
MAE
엠에이이
나이 예측 핵심

Mean Absolute Error, 즉 평균 절대 오차입니다.

  • 실제 나이와 예측 나이 차이의 평균
  • MAE 6세는 평균적으로 약 6세 차이난다는 뜻
  • 사용자 체감에 가장 가까운 나이 예측 지표
RMSE
알엠에스이
평가 지표

Root Mean Squared Error, 큰 오차에 더 민감한 지표입니다.

  • 한두 장을 크게 틀리면 RMSE가 크게 올라감
  • MAE는 평균 체감 오차, RMSE는 큰 실패 위험 확인용
Accuracy
애큐러시 / 정확도
분류 지표

성별 분류처럼 정답이 있는 분류 문제에서 맞힌 비율입니다.

  • 88.19%면 검증 이미지 중 약 88.19%를 맞힌 것
  • 데이터 불균형이 있으면 Accuracy만으로는 부족할 수 있음
Overfitting
오버피팅 / 과적합
주의

모델이 학습 데이터는 잘 맞추지만 새로운 사진에는 약해지는 현상입니다.

  • Train MAE는 낮아지는데 Val MAE가 오르면 의심
  • Early stopping으로 어느 정도 방지
Calibration
캘리브레이션 / 보정
오차 개선 후보

모델 예측값이 특정 나이대에서 치우칠 때 결과를 후처리로 조정하는 방법입니다.

  • 예: 60대 이상을 낮게 예측하면 고령층 보정 적용 가능
  • 새 학습 없이 체감 오차를 줄이는 데 도움

나이 예측 모델

ResNet18
레즈넷 에이틴
Baseline

Residual Network 18. 안정적이고 비교적 가벼운 CNN 모델입니다.

  • Test MAE 6.1019세
  • 기준 모델로 의미가 있지만 최종 후보에서는 제외
MobileNetV3
모바일넷 브이쓰리
경량 모델

모바일/배포 환경을 고려한 가벼운 CNN 모델입니다.

  • Best Val MAE 6.2679세
  • 빠르지만 ConvNeXt-Tiny보다 정확도가 낮음
ConvNeXt-Tiny
컨브넥스트 타이니
원본 베이스

CNN을 현대적으로 개선한 모델로, 현재 프로젝트 나이 예측의 최종 베이스 모델입니다.

  • Best Val MAE 5.9904세
  • Best Epoch 7
  • 이 모델을 추가 fine-tuning하여 최종 후보를 만들었음
ConvNeXt-Tiny Fine-tuned
컨브넥스트 타이니 파인튜닝
최종 후보

기존 ConvNeXt-Tiny best 모델을 낮은 학습률로 추가 튜닝한 최종 나이 예측 모델입니다.

  • Fine-tuned Best Val MAE 5.6993세
  • 최종 Test MAE 5.4106세
  • RMSE 7.4872세
  • 5세 이내 비율 59.61%, 10세 이내 비율 84.30%
  • 재학습 전 Test MAE 5.7122세보다 약 0.3016세 개선
EfficientNet-B0
이피션트넷 비제로
비교 모델

정확도와 효율의 균형을 맞춘 CNN 모델입니다.

  • Best Val MAE 6.1760세
  • MobileNetV3보다 좋지만 ConvNeXt-Tiny보다 낮음

성별 분류 모델

MobileNetV3-Small Gender Classifier
모바일넷 브이쓰리 스몰 젠더 클래시파이어
최종 후보

성별 분류를 위해 학습한 경량 모델입니다. 성별 분류는 나이 예측보다 단순한 이진 분류라 가벼운 모델이 적합합니다.

  • Best Val Accuracy 88.19%
  • Best Epoch 5
  • 최종 앱에서 성별 안내/미래 얼굴 처리 보조 정보로 사용 가능

미래 얼굴 모델

SAM
샘
현재 후보

Style-based Age Manipulation. 입력 얼굴을 target age 조건에 맞춰 변환하는 얼굴 에이징 모델입니다.

  • 가중치: `sam_ffhq_aging.pt`
  • 로컬 실행 성공
  • 외국인 중심 데이터 학습이라 한국인 정체성 유지 한계 있음
  • 최종 표현은 “미래 예측”보다 “재미용 시뮬레이션”이 적절
StyleGAN2 / Latent Space
스타일갠 투 / 레이턴트 스페이스
생성 모델

SAM 내부에서 얼굴 이미지를 생성하고 조작하는 기반 개념입니다.

  • Latent space는 얼굴 특징이 압축된 내부 표현 공간
  • 나이 방향으로 latent를 조작하면 나이 든 얼굴을 생성 가능
FFHQ
에프에프에이치큐
한계 원인

Flickr-Faces-HQ. 고품질 얼굴 이미지 데이터셋입니다.

  • SAM 공개 가중치의 기반 데이터와 관련
  • 한국인 중심 데이터가 아니므로 결과가 서양인화될 수 있음
LATS
래츠 / 엘에이티에스
제외

Lifespan Age Transformation Synthesis. 얼굴 사진을 여러 나이대로 변환하는 후보 모델이었습니다.

  • 테스트에서 한국인 얼굴이 서양인처럼 변함
  • 여성 얼굴에 원치 않는 수염/성별 변화 느낌 발생
  • 피부가 자연스럽게 늙기보다 붕괴되는 느낌
  • 최종 생성기에서 제외
CUSP
커스프
보류

Custom Structure Preservation in Face Aging. 얼굴 구조 보존을 강조하는 face aging 연구 모델입니다.

  • 코드 확보는 가능했지만 공식 가중치 접근이 어려웠음
